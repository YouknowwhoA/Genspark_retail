Core Requirements

1. Data Layer

- Download and load the Kaggle Retail Transaction Dataset

- Store data in a database or file-based system (your choice)

- Implement data access layer (a RPC or REST API) to query:

- Customer-specific information (by CustomerID)

- Product-specific information (by ProductID)

- Aggregated metrics for business intelligence

2. LLM Integration

- Integrate any LLM (ChatGPT API, local models like Ollama, etc.)

- Implement a system that:

- Parses user intent (identify if asking about customer, product, or business metrics)

- Extracts relevant IDs or parameters from natural language queries

- Retrieves appropriate data from the dataset

- Generates natural language responses based on data

3. Chat Interface

- Create a chat-based UI for querying the system

- Minimal requirement: these types of queries:

- A. Customer-Specific Queries (input: customer_id)

- Examples:

- "What is the purchase history for customer C67890?"

- "How much has customer C11111 spent in total?" B. Product-Specific Queries (input: product_id)

- Examples:

- "What's the average discount for product P1234?"

- "Which stores sell the product P9999?"

4. Query Processing Logic

- Implement intent classification to route queries appropriately

- Extract entities (customer IDs, product IDs) from natural language

- Execute appropriate SQL/data queries based on user intent

- Handle edge cases (invalid IDs, ambiguous queries, no data found)

5. Stretch goal: Enhance query flexibility to support a broader range of scenarios and unlock more use cases.

Technical Stack Suggestions

- Frontend: Vue.js, React, or Streamlit

- Backend: Python (FastAPI/Flask), Node.js, or Go

- Database: SQLite, PostgreSQL, MySQL, or Pandas DataFrames

- LLM: OpenAI API, Anthropic Claude, Ollama (local), or any other LLM

Note: These are suggestions only. Use any technology stack you're comfortable with.

Testing Requirements

- End-to-end testing for core query flows:

- Customer query → data retrieval → response generation

- Product query → data retrieval → response generation

- Business question → aggregation → response generation

- Unit tests for core logic, no need high coverage
